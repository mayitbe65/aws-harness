DEPLOYMENT WORKFLOW
═══════════════════════════════════════════════════════════════════════════════

Step 1: SETUP
  ├─ npm install (dependencies)
  ├─ cp .env.example .env (configuration)
  └─ Edit .env with AWS account ID

Step 2: VERIFY
  ├─ aws sts get-caller-identity (credentials)
  └─ npm run build (TypeScript compilation)

Step 3: BUILD DOCKER IMAGES
  ├─ ./scripts/build-docker.sh dev
  ├─ Creates ECR repositories (auto)
  ├─ Builds backend image from Python
  └─ Builds frontend image from Node.js

Step 4: PREVIEW
  └─ npm run cdk:diff (show changes)

Step 5: DEPLOY
  └─ npm run cdk:deploy:dev
     (Creates entire infrastructure)

Step 6: VERIFY
  ├─ CloudFormation console
  ├─ ECS cluster status
  ├─ RDS instance details
  └─ Frontend and backend URLs

QUICK START COMMANDS
═══════════════════════════════════════════════════════════════════════════════

# Setup and Install
cd /workshop/aws-harness/cdk
npm install
cp .env.example .env
# Edit .env with your AWS account ID

# Build and Deploy
./scripts/build-docker.sh dev
npm run cdk:deploy:dev

# Monitor
aws logs tail /ecs/dev/errorqa-backend --follow

# Get Output URLs
aws cloudformation describe-stacks \
  --stack-name ErrorQaStackDev \
  --query 'Stacks[0].Outputs'

# Destroy (⚠️  WARNING)
npm run cdk:destroy:dev
